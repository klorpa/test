・使用条件
	デバッグモードONのときにダンジョン探索を選ぶと[99]が現れるのでそれを選択
	選択すると10*10の■が現れます
	これがMAPに相当します

・現在の階層
	現Mに相当します
	ひとつのダンジョンは24枚のMAPまで持てるので0~24の値を持ちます
・最大X・最大Y
	MAPのサイズを規定しており、右のボタンで増減させられます
	両方共最低値は1で、最大値は20です
・ライン番号
	MAPの上と左についている、水色と黄緑色の数字はそれぞれX座標とY座標を表しています
	この数字をクリックすることで、クリックした列と行の色を変えて強調できます
	強調を消すときはすでに強調されている行を再度クリックしてください
	なお、強調が重なった場合は紫色になり、後述するイベントが設置されている場合は赤色が優先されます
・設置マス選択
	左のタイル設置・イベント設置をクリックすることで
	タイル設置モードとイベント設置モードが切り替わります

・タイル設置モード
	下の9種類の文字をクリックすると、クリックした文字が赤くなります
	この状態で、MAPの任意の座標をクリックすることでその座標の文字が赤い文字のものに置き換わります
	また、選択中のタイルと、座標に設置されているタイルが同じ場合、タイルは通路に置き換えられます
	なお、罠のみ正常に機能せず、特殊タイルは特でひとくくりにしかできません
・イベント設置モード
	下の20個の数字（現在階層4のみ19個）をクリックすることで、タイル設置を同じく赤くなります。
	このときタイル設置とおなじく、任意の座標クリックでその座標に数字をおくことができます
	ただし、イベントはおなじ数字をひとつしかおけませんので、注意してください
	また、すでに数字がおかれている座標をクリックした場合、削除するかどうか聞かれます
	・イベント設置
		イベント設置をする際に、イベントの内容を聞かれます
		現在は5種を設定することができます
	・宝箱
		　アイテム名・使用フラグ・個数（価格）を設定できます
		　アイテム名は文字列を入力できます.
		　宝箱に置きたいもののメモとして名前をつけておくか
		　直接、いれるものアイテムナンバーを入力しておくといいです
		　このアイテム名を魔貨・円にした場合、MONEY入手のイベントになります
		　使用フラグは宝箱をあけたら消えるようにするためのフラグです
		　DB:(FLAG:現在のダンジョン):Xのうち、Xの部分を決めることが出来ます
		　ほかの使用フラグとのかぶりに注意しつつ,設定しましょう
		　個数（価格）は読んで字の通りで、アイテムの場合は入手個数、円・魔貨の場合は価格になります
	・階段、ワープゾーン
		　とぶ先のM/X/Yを設定できます
		　とぶ先のM/X/Yはあらかまじめ確認しておいてください
	・その他イベント
		　文字列を入力しておくことができます
		　ボス戦や入り口やターミナルなど、イベント内容を軽く書いとくといいとおもいます
	なお、イベント設置モードで設置したイベント番号はタイル設置モードでは消えますが
	その代わり、イベントが設置されているマスは赤くなります
・フラグリスト
	ざっと、フラグの内容を書き出します
	現在フラグは100個まで使える仕様なので0~99のうち、設置されているものを書き出していきます
・アウトプット
	現在表示しているMAPをERBに記述出来る形で書き出していきます。
	Emueraのメニューを開き、ログをクリップボードにコピーするを選ぶか、ログを保存するを選ぶことで
	書き出したものをコピー出来るようになるので、新ダンジョンのERBにペーストしてやってください。
	また、このとき設置した壁と通路以外のタイルと座標と名前、設置したイベントの内容も書き出し、
	命令を書ける場合は命令も書き出すので、実際にイベントを組む際の参考にしてください
	上記ふたつの書き込みが終わったあと、読み込み用のMAPも書き出すか、選択できます。
	・読み込み用のMAP
		現在、未実装ですが、仕様的に外部に書き出さなくてもMAPを記憶しておけるので、実はなくても大丈夫
・インプット
	外部のファイルからMAPを読み込み、その配置を再現する機能です
	読み込み先はすでに作成されているダンジョンと読み込み専用の関数からの二つがあります。
	・すでに作成されているダンジョン
		＊現在未実装です。
		ガイドに従って、ダンジョン番号と階層を選択してください
		ダンジョンがない場合は警告がでますが、階層が存在しない場合はすべて空白のMAPを吐き出すので注意してください
		タイル設置だけ再現するだけで、イベントはまったく設置されていない状況になるのでそれも注意です
		また現在階層0にしか呼び出せず、上書きするのにも注意してください
	・読み込み専用関数
	　DUNGEONMAKER_INPUT.ERBにある関数@INPUT_DUNGEON_X(ARG)からダンジョンを読み込みます
	　とりあえず、10000まで選べるようにしておきました

	　ただまぁ一応、メーカー自体がデバッグ用につかえるようなきもするので
	　ダンジョンを作成してアップする際にデバッグ向けに読み込み専用MAPもUPしたいという場合は
	　ダンジョン番号*100＋階層の数値で、関数をつくるといいとおもいます
	　例:ゾディアックの階層1なら、13*100+1で、INPUT_DUNGEON_1301(ARG)
