A.構想
	1.全ソースと珠習得を主人にも解放し、奴隷とおなじ成長方法にする
		でも、欲情とか快感はともかく、恭順とか屈従とか恐怖とか主人がとるとおかしすぎるので
		このへんは置き換え考えないとだめそう
	
	2.全コマンドに調教者ソースと非調教者ソースを設定し、奉仕と愛撫を一元化
		まぁつまり愛撫だと、調教者は習得系のソース、非調教は快感系のソースを得るとか
		正常位だと、調教者は快Ｃ、非調教者は快∨を得るとか
	
		また、これと同時に奴隷も特殊なものを除いた、すべてのコマンドの行為者になることができ、
		主人もすべてのコマンドの被対象者になれるようにデザインする

	3.ソース処理を奴隷・主人・助手で3回やる
		上記で拡張したソースを使うためにソース処理は3人とも行う感じに
		PLAYERとTARGETじゃないのは3Pとか、ダブルフェラは3人ともソースを得ることになるから
		このとき、PLAYER:1を奴隷に対する行為者、PLAYER:2を主人に対する行為者、PLAYER:3を助手に対する行為者とし
		行為者を判定に使う場合に利用する

	4.奴隷に調教者にたいして、コマンドを実行させられる機構を作る
		理想は「身をゆだねる」とかつくって、奴隷がＡＩにそって、ランダムでコマンドを選択し
		行為に対して、褒めるとか怒るとか、そんな反応を返して、ＡＩに学習させていく

		けどまぁ、そこら辺組むのはやっかいすぎるし、主導権渡した時点で、行為者を奴隷として、実行させるくらいに落ち着きそう
